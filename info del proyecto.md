1. El sistema debe permitir visualizar un catálogo de productos organizado por categorías (hombre, mujer y niños), mostrando imagen, nombre y precio.

2. El sistema debe permitir a los usuarios buscar y filtrar productos por categoría, talla, rango de precios y tipo de prenda mediante herramientas dinámicas.

3. El sistema debe mostrar una vista detallada de cada producto, incluyendo múltiples imágenes, descripción, tallas disponibles, precio y disponibilidad en stock.

4. El sistema debe permitir al usuario seleccionar talla y cantidad, y agregar productos al carrito de compras.

5. El sistema debe permitir al usuario gestionar el carrito de compras, incluyendo modificar cantidades, eliminar productos y visualizar el total de la compra.

6. El sistema debe permitir a los usuarios registrarse e iniciar sesión, mediante el ingreso de datos personales y credenciales.

7. El sistema debe permitir a los usuarios autenticados guardar productos en una lista de favoritos.

8. El sistema debe permitir a los usuarios realizar pedidos en línea, ingresando datos de entrega como dirección, teléfono y referencias.

9. El sistema debe permitir al administrador gestionar productos, incluyendo registrar, editar y eliminar productos con sus respectivos datos (imágenes, descripción, precio, tallas y stock).

10. El sistema debe permitir al administrador gestionar el inventario, actualizando las cantidades disponibles de cada producto.

11. El sistema debe permitir al administrador gestionar pedidos, visualizando y actualizando estados como pendiente, confirmado, enviado o entregado.

12. El sistema debe mostrar en la página principal productos destacados, promociones y nuevas colecciones, actualizables por el administrador.

13. El sistema debe permitir la búsqueda de productos con visualización de métricas de ventas, mostrando la cantidad total de unidades vendidas por cada producto.

14. El sistema debe permitir a los usuarios autenticados visualizar su historial de pedidos, incluyendo detalles como productos adquiridos, fechas, cantidades y estado de cada pedido.

15. El sistema debe permitir al administrador generar reportes de ventas, filtrando la cantidad de ventas realizadas por mes y mostrando las ganancias obtenidas en cada periodo.
